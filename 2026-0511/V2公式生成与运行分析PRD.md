# V2公式生成与运行分析 PRD

## 1. 总纲

| 原则 | 说明 |
|------|------|
| 核心目标 | 以收益率为核心目标 |
| B/S配对 | 历史第一个必须是B点，B/S必须成对出现，B和S之间不能再出现B |
| 买卖分离 | 买入算法和卖出算法分开，不能相同逻辑 |
| 性能 | 需要考虑计算性能 |
| 个性化 | 每只股票操盘手不同，买卖指标各不相同，需针对每只股票分析 |
| 最低持仓 | 1个交易日 |
| 持久化 | 权重和B/S点按股票代码持久化，支持重新生成和删除 |
| 扩展性 | 规则和条件层需考虑后续新增扩展 |

## 2. 整体架构

### 2.1 两大层

```
第一大层：计算各买卖指标有效性
  ├── 提取S数组（历史高点集合）
  ├── 提取B数组（历史低点集合）
  ├── 循环S数组 → 卖出规则匹配 → 奖励值+1
  └── 循环B数组 → 买入规则匹配 → 奖励值+1

第二大层：迭代优化（最大100次循环）
  ├── 模块1：B/S打标 + 有效性验证
  │   ├── 用模块2的权重打标B/S点
  │   ├── B点有效性：买入后到下一个S点间最高价利润
  │   └── S点有效性：卖出后到下一个B点间最低价避免回撤
  └── 模块2：最终权重计算
      └── 利用模块1的奖励/惩罚值计算归一化权重
```

### 2.2 分析功能

```
分析功能
  ├── 用生成的权重+充分必要条件打标B/S点
  ├── 支持新要求在K线上确认B/S点
  └── 充分必要条件展示（如B点需等1天阳线确认）
```

## 3. 第一大层：计算各买卖指标有效性

### 3.1 数据预处理

- 去除历史最开始10个交易日
- 去除最近5个交易日
- 去除价格小于0的数据
- 结果作为后续所有买卖规则权重数据分析的来源

### 3.2 S数组提取（历史高点集合）

**目标**：找出所有历史高点，作为卖出规则验证的锚点

**算法**：
1. 获取历史最高点S1
2. 寻找第二高点S2，需满足：
   - S1与S2相隔日期绝对值 ≥ 5
   - S1与S2之间所有天的价格连线**是**开口向上的抛物线（若不是则舍弃）
3. 继续寻找后续高点，同样规则
4. 终止条件：找完所有高点，或集合满足50个
5. 结果S[]作为后续分析点

**抛物线判断**：
- 对S1到S2之间的收盘价做二次拟合 `y = ax² + bx + c`
- 若 `a > 0`（开口向上），则保留该点
- 若 `a ≤ 0`（开口向下或直线），则舍弃

### 3.3 B数组提取（历史低点集合）

**目标**：找出所有历史低点，作为买入规则验证的锚点

**算法**：
1. 获取历史最低点B1
2. 寻找第二低点B2，需满足：
   - B1与B2相隔日期绝对值 ≥ 5
   - B1与B2之间所有天的价格连线**不是**开口向下的抛物线（若是则舍弃）
3. 继续寻找后续低点，同样规则
4. 终止条件：找完所有低点，或集合满足50个
5. 结果B[]作为后续分析点

**抛物线判断**：
- 对B1到B2之间的收盘价做二次拟合 `y = ax² + bx + c`
- 若 `a < 0`（开口向下），则舍弃该点
- 若 `a ≥ 0`（开口向上或直线），则保留

### 3.4 卖出规则有效性验证

对S数组中每个S点：
- 寻找其日K线前后90天窗口
- 用所有卖出规则验算有效性
- 匹配到则在对应规则的**奖励值**上加1
- 注意规则本身参数合理性（如M头交易日天数需合理）

### 3.5 买入规则有效性验证

对B数组中每个B点：
- 寻找其日K线前后90天窗口
- 用所有买入规则验算有效性
- 匹配到则在对应规则的**奖励值**上加1
- 注意规则本身参数合理性（如W底交易日天数需合理）

### 3.6 输出

每个规则的初始奖励值和惩罚值都是0，经过第一层后：
```
{
  'buy_rules': {
    'W底突破': {'reward': 15, 'penalty': 0},
    'V反底部': {'reward': 8, 'penalty': 0},
    ...
  },
  'sell_rules': {
    'M顶跌破': {'reward': 12, 'penalty': 0},
    'V反顶部': {'reward': 10, 'penalty': 0},
    ...
  }
}
```

## 4. 第二大层：迭代优化

### 4.0 核心原则

- **不可偷看未来**：打标B/S点时，不能使用当天之后的数据
- **验算可用未来**：B/S点确定后，验算有效性时可查看之后数据
- **买卖有效性必须在历史K线全部打标之后再验算**

### 4.1 模块1：B/S打标 + 有效性验证

#### 4.1.1 B/S点打标逻辑

按照模块2计算的权重，用历史K线逐日打标B/S点：
- 必须注意规则的时间跨度合理性（如向前90天验证M头和W底）
- 第一个信号必须是B
- B/S必须成对出现
- 最低持仓1个交易日

**打标流程**：
```
for idx in range(start, end):
    if 当前无持仓:
        检查买入条件（必要条件 + 充分条件 + 关键形态）
        满足 → pending_buy，等下一天阳线确认
    if 当前有持仓:
        检查卖出条件（必要条件 + 充分条件 + 关键形态/必卖）
        满足 → 标记S点
```

#### 4.1.2 B点有效性验证

以买入后到下一个S点的价格最高日的利润来计算：

| 收益区间 | 奖惩 |
|----------|------|
| 收益 < 0 | 触发规则的惩罚值 +1 |
| 0 ≤ 收益 < 10% | 触发规则的奖励值 +1 |
| 10% ≤ 收益 < 20% | 触发规则的奖励值 +2 |
| 20% ≤ 收益 < 30% | 触发规则的奖励值 +3 |
| ... | 依次类推 |

**收益计算**：`收益 = (BS之间最高价格 - B点价格) / B点价格 × 100%`

#### 4.1.3 S点有效性验证

以卖出后到下一个B点的价格最低日的避免利润回撤来计算：

| 避免回撤区间 | 奖惩 |
|-------------|------|
| 避免回撤 < 0 | 触发规则的惩罚值 +1 |
| 0 ≤ 避免回撤 < 10% | 触发规则的奖励值 +1 |
| 10% ≤ 避免回撤 < 20% | 触发规则的奖励值 +2 |
| ... | 依次类推 |

**避免回撤计算**：`避免回撤 = (S点价格 - SB之间最低价格) / S点价格 × 100%`

### 4.2 模块2：最终权重计算

利用模块1最终计算的各规则奖励值和惩罚值：

**权重公式**：
```
原始权重 = reward - penalty
有效权重 = max(原始权重, 0)  # 负权重置0，无效规则不参与
归一化权重 = 有效权重 / Σ(所有有效权重)  # 按买入/卖出分组归一化
```

**胜率计算**：
```
胜率 = reward / (reward + penalty)  # 规则触发的成功率
```

**综合评分**（用于规则排序和筛选）：
```
综合评分 = 归一化权重 × 胜率
```

**规则分级**：
| 级别 | 条件 |
|------|------|
| strong | 综合评分 ≥ 0.15 |
| medium | 0.05 ≤ 综合评分 < 0.15 |
| weak | 综合评分 < 0.05（不参与打标） |

### 4.3 迭代终止条件

- 最大循环次数：100次
- 收敛条件：连续3次迭代收益率变化 < 0.1%
- 取成功率和收益率最大的结果

## 5. 分析功能

### 5.1 B/S点打标

- 按照生成的规则权重以及充分必要条件，在K线上打标B/S点
- 最终B/S点按股票代码持久化到MongoDB

### 5.2 充分必要条件

#### 5.2.1 买入必要条件（必须全部满足才可买入）

| 条件 | 说明 |
|------|------|
| 非阴线 | 当日收盘价 > 开盘价 |
| 突破5日线 | 当日收盘价 > MA5 |
| 阳线确认 | 买入信号后等1天，下一天为阳线才确认B点 |
| 非下跌趋势 | MA20上升或价格 > MA20 |

#### 5.2.2 买入充分条件（满足任一即可触发买入候选）

| 类型 | 条件 |
|------|------|
| 关键形态必买 | W底突破、头肩底突破、V反底部、旗形突破 |
| 加权买入 | buy_score > sell_score 且 buy_score > 0 |

#### 5.2.3 卖出必要条件（必须满足才可卖出）

| 条件 | 说明 |
|------|------|
| 当前持仓 | 最近一个信号为B |

#### 5.2.4 卖出充分条件（满足任一即可触发卖出）

| 类型 | 条件 |
|------|------|
| 关键形态必卖 | M顶跌破、V反顶部、旗形跌破、均线趋势打破必卖 |
| 计数卖出 | ≥2条卖出规则触发且sell_score > buy_score |

### 5.3 条件展示

页面需展示所有充分必要条件，如：
- B点需等1天阳线确认
- S点关键形态直接触发
- 各条件的详细参数说明

## 9. 性能优化

1. **规则预计算**：所有规则信号预先计算为布尔数组，避免逐日重复调用
2. **向量化操作**：使用numpy向量化代替循环
3. **增量计算**：权重已存在时跳过第一层，直接进入分析
4. **缓存机制**：权重和B/S点缓存到MongoDB，避免重复计算
