# V5 · V 左几何识别（因果）

> **状态**：几何识别已实现（`backend/v4_aggressive/v_left_geometry.py`），待迁入 `backend/v5/` 并接入买卖规则。  
> **关联**：[V5 B/S 规则说明（草案）](./V5-BS规则说明.md) — 本文只讲 **V 左识别**，不涉及右侧与 B/S 信号。

---

## 1. 设计目标

| 原则 | 说明 |
|------|------|
| **因果** | bar `t` 只使用 `[0..t]` 数据；标注 V 左时不偷看右侧、不要求 V 右已走出 |
| **确认锁定** | V 底 bar `b` 在后续 bar `t` 出现 ATR 级反弹后锁定；**已锁定标注不随后续 K 线回溯修改** |
| **非固定 K 数** | 左肩宽度、跌段长度用 **ATR 路径 / 跌深比例** 衡量，不用「必须 N 根 K」 |
| **结构优先** | V 左 = 多 bar 的下跌 **过程**；单 K 线 high→low、浅跌 event 一律否 |
| **可收敛** | 三层固定架构 + ATR 倍数参数；避免在单股上堆 if/else 补丁 |
| **多 leg** | 同一宏观下跌中，若中间有显著反弹，可识别 **多个独立 V 左** |
| **统一名词** | 阻力统称 **压力位**（含 **静态锁定** + **持仓期动态** 如 MA20，见第 9 节） |

---

## 2. 三层架构

```
┌─────────────────────────────────────────────────────────┐
│ Layer 1  结构不变量（硬约束，不可绕过）                    │
├─────────────────────────────────────────────────────────┤
│ Layer 2  去噪（ATR 窗口阶段新低才评估）                    │
├─────────────────────────────────────────────────────────┤
│ Layer 3  在线确认锁定（标注不可回溯修改）                  │
└─────────────────────────────────────────────────────────┘
```

**用户语义（示例）**：bar `n` 为左肩高点，bar `n+8` 为 V 底，bar `n+10` 发现反弹可确认 → 标注 **V 左 = n 日～n+8 日**；继续往后走时 **不得删除或改写** 该标注（即使之后出现更深底）。

### Layer 1 — 结构不变量

以下 **全部满足** 才可能为 `v_left`：

1. **左肩 bar 早于 V 底 bar**（`bar_span ≥ 1`）  
   - 禁止同一根 K 的 intraday high→low 冒充左段  
2. **左肩→V 底 ATR 路径 ≥ 1.5**  
   - 左段必须是「走过」的价格过程  
3. **幅度门槛**：跌深 ≥ 10% **或** ATR 路径 ≥ 2.2  

违反 Layer 1 → `v_left_event_fake`（event 假 V）或 `v_left_weak`。

### Layer 2 — 去噪

仅在 **ATR 窗口内创阶段新低** 的 bar 上调用评估函数：

- 从 `t` 向左累计 ATR 路径 ≥ 1.5，得窗口 `[start..t]`
- 要求 `low[t]` 为该窗口内最低  
- 滤掉跌途中的小幅毛刺，减少「天天是 V 底」

### Layer 3 — 在线确认锁定

逐 bar 推进（模拟实盘），**locked 列表只增不改**：

1. **候选**：bar `b` 为 ATR 阶段新低且 Layer 1 通过 → 加入 `pending`
2. **作废**：若之后 `low[t] < low[b]`（底被破）→ 从 `pending` 移除（未确认前）
3. **确认**：bar `t` 上，自 `b` 起反弹 `≥ V_LEFT_CONFIRM_BOUNCE_ATR × ATR[b]` → 锁定 V 左 `[peak..b]`，`confirm_idx = t`
4. **不可变**：已锁定项不随后续更深底、同 leg 去重而删除

几何参数（peak、drop、atr_path）在 **V 底日 `b`** 用 `[0..b]` 计算；确认只用 `[b+1..t]` 的反弹幅度。

| 参数 | 默认值 | 含义 |
|------|--------|------|
| `V_LEFT_CONFIRM_BOUNCE_ATR` | 0.75 | V 底后反弹确认阈值 |
| `V_LEFT_CONFIRM_MIN_BARS` | 1 | 确认前 V 底后至少经过 bar 数 |

## 3. 跌段 leg 分段（多 V 左）

**问题**：一次宏观下跌 + 中间反弹，应标多个 V 左（如 002979 的 2024-12 两段）。

**规则**（因果）：

从 V 底 `t` 向左扫描前一次谷底 `i`：

- 若 `low[t] < low[i]`（当前更深）
- 且 `max(high[i+1..t]) - low[i] ≥ LEG_BOUNCE_ATR × ATR[i]`（中间反弹够大）

→ 当前 leg 从反弹高点之后重新开始；**左肩只在本 leg 内**取 `high` 最大点（且严格早于 `t`）。

| 参数 | 默认值 | 含义 |
|------|--------|------|
| `LEG_BOUNCE_ATR` | 1.0 | 两跌段分界：前底到后续反弹高点 ≥ 此 ATR 倍数 |

---

## 4. 左肩与度量

| 字段 | 定义 |
|------|------|
| `effective_peak_idx` | 当前 leg 内、严格早于 V 底的 **high 最大点** |
| `leg_start_idx` | leg 起点（上次 ATR 级反弹后的反弹峰位置） |
| `drop_pct` | `(high[peak] - low[bottom]) / high[peak]` |
| `atr_path` | 左肩→V 底，均价曲线逐步累计的 ATR 归一化路径 |
| `bar_span` | `bottom_idx - effective_peak_idx` |
| `bottom_rejection` | V 底 K 线(close-low)/(high-low)，探底拒绝程度 |
| `path_efficiency` η | 净跌 / 路径长度，越接近 1 越「直跌」 |

### 分类

| kind | 含义 |
|------|------|
| `v_left` | V 左有效，可用于 V5 规则 |
| `v_left_event_fake` | event 假 V（单 K、极短 ATR、浅跌等） |
| `v_left_weak` | 结构边缘，暂不用于信号 |

## 9. 压力位（V5 全场景统一阻力标注）

> **2025-05-19 修订**：压力位 **不限于 V**。W、横盘、非标准 V、N 字、上方缺口、摆动高点等 **一切** 上方阻力，对外 **只称「压力位」**。  
> 旧称（瓶口、历史前高、neck、阻力带…）**停用**；实现里可用来源码区分，不对用户展示。  
> **本文档记录每次规则变动。**

### 9.1 大原则

| # | 原则 |
|---|------|
| 1 | **因果**：bar `t` 只用 `[0..t]`，不偷看将来 |
| 2 | **等待**：当日判不出 → 不标，等下一根 K |
| 3 | **统一名词**：界面与规则只出现「压力位」 |
| 4 | **两类压力位**（对外同名，内部用 `mode` 区分，见 9.2） |

**静态（锁定）压力位** — 结构/摆动/缺口等：

| # | 原则 |
|---|------|
| S1 | `t_c` 确认后，`P`、压力带、锚点 bar **永久不变** |
| S2 | 只增不改：可新增更高处压力位；**不得** 改写已锁定条目 |

**动态压力位** — 均线等随行情每日变化：

| # | 原则 |
|---|------|
| D1 | **仅在持仓期间** 计算与标注（有仓即从开仓 bar 起；**不限** 是哪一种买入规则触发的 B） |
| D2 | bar `t` 当日用 `[0..t]` 算出的 `P_t`（如 MA20[t]），**该日一旦落库不再修改** |
| D3 | bar `t+1` 的 `P_{t+1}` 可不同于 `P_t`（动态上移/下移），但 **不得回头改** `P_t` |
| D4 | **空仓** 不输出动态压力位（避免无持仓时的均线干扰） |

**「B」在 V5 中的含义（已定）**  
- 用户语境里的 **B = 持仓**，不是某个固定规则名。  
- 只要回测/实盘判定 **当前有持仓**（`position_open`），即进入动态压力计算区间。  
- 开仓 bar = `open_idx`（由成交/信号逻辑确定，与 V5 买入规则种类无关）。  
- 平仓 bar = `close_idx`（S 或止损等清仓日）；平仓后该段动态压力停止。

**示例（动态 + 因果）**  
- bar `open` 建仓 → 从 `open` 起每个持仓日记录 MA20 压力位  
- bar `open+2`：`P = MA20[open+2]`（只用至该日收盘）并锁定该日快照  
- bar `open+10`：`P = MA20[open+10]`，**不会** 回头修改 open+2 那天的值  

### 9.2 两类实体（对外均称「压力位」）

| `mode` | 名称（对外） | 典型来源 | P 是否随日变化 |
|--------|--------------|----------|----------------|
| `locked` | 压力位 | V左肩、W颈线、横盘顶、缺口、摆动高点… | 否，确认后固定 |
| `dynamic` | 压力位 | MA20、MA60、EMA… | 是，**每日一条快照** |

**静态 `locked` 字段**

| 字段 | 含义 |
|------|------|
| `anchor_idx` | 锚点 bar `h` |
| `P` | 压力中心价 |
| `zone` | 压力带，锁定后固定 |
| `confirm_idx` | 确认锁定日 `t_c` |
| `broken_idx` | 可选；首次突破 bar（追加） |

**动态 `dynamic` 字段**

| 字段 | 含义 |
|------|------|
| `bar_idx` | 本条快照所属 bar `t`（须在持仓区间内） |
| `P` | 当日压力价，如 `MA20[t]`（仅用 `[0..t]`） |
| `zone` | 可选，如 `[MA − band, MA + band]`，当日固定 |
| `open_idx` | 本段持仓的开仓 bar（有仓起点） |
| `close_idx` | 本段持仓的平仓 bar；平仓后停止该段动态压力 |
| `indicator` | 内部码：`ma20` / `ma60` / …（不对用户展示别名） |

同一 bar `t` 上可同时存在：多条 `locked` + 一条或多条 `dynamic`（不同均线）。

### 9.3 架构：静态多来源 + 持仓期动态序列

```
【静态 locked】
摆动高点 · V左肩 · W颈线 · 横盘顶 · N字前高 · 缺口 · M顶 …
        → 候选 → 确认门 → locked 注册表（只增不改）

【动态 dynamic】
持仓 @ open  →  对每个持仓日 t（open ≤ t ≤ close）：用 [0..t] 算 MA20[t] 等
        →  写入 (position_id, t, P_t) 日快照（P_t 不可回溯修改）
```

- **静态探测器**：结构/摆动/缺口 → 确认 → **locked**  
- **动态计算器**：bar `t` 在 **持仓状态** 且 MA 可算 → **当日 dynamic 快照**  
- **注册表**：`locked[]` + `dynamic_series[(position_id, indicator)][]`  
- **持仓判定 @ t**：仅用 `[0..t]` 的已成交买卖/仓位状态（因果），与具体 B 规则名无关

### 9.4 场景 → P 怎么取（内部来源码，对外均称压力位）

| 来源（仅实现/日志） | P 怎么取 | 候选（因果） | 确认（草案） |
|---------------------|----------|--------------|--------------|
| 摆动高点 | `high[h]` | 阶段新高 @ h | 自 h 回落 ≥ ATR |
| V / 非标准 V 左肩 | 已锁 V 左的 peak high | V 左 pending/锁定 | 左肩后回落 或 与 V 左同日结构确认 |
| W 颈线 | 两底间 high 最大 | 第二底 + 颈线可识别 | W 右侧结构确认日 |
| 横盘上沿 | 箱体内 high 最大 | 横盘在 [0..t] 成立 | 触顶回落 或 下破前锁顶 |
| N 字前高 | 右侧 leg 内 high 最大 | V/W 右侧 + leg 可识别 | 回踩触压回落 |
| 上方缺口 | 缺口上沿（左棒 low） | 跳空形成日 g | 形成即锁 或 g+k 仍未回补 |
| M 顶 / 双顶 | 较高峰或颈线区 high | 双顶在 [0..t] 可识别 | 颈线跌破或峰后回落 |

同价多来源 → **合并一条** 压力位（先锁优先）。  
**无 V 左** 也可有压力位（横盘顶、缺口等）。

### 9.5 确认门（两类）

**A. 价格确认**（摆动高点、V 左肩、N 字前高等）  
在 bar `t`：`high[t]` 未破 P，且 `high[h] − min(low[h..t]) ≥ PRESSURE_CONFIRM_DROP_ATR × ATR[h]` → 锁定。

**B. 结构确认**（W 右侧、缺口、部分横盘）  
在 bar `t_c`，结构在 `[0..t_c]` 已成立 → 锁定表中 P（定义内若含回落则已包含）。

### 9.6 压力带

```
band = max(PRESSURE_BAND_ATR × ATR[h], PRESSURE_BAND_MIN_PCT × P)
zone = [P − band, P + band]   # 缺口可用 [gap_bottom, gap_top] 一次固定
```

| 参数 | 默认 | 含义 |
|------|------|------|
| `PRESSURE_CONFIRM_DROP_ATR` | 0.75 | 价格确认回落 |
| `PRESSURE_BAND_ATR` | 0.5 | 带半宽 |
| `PRESSURE_BAND_MIN_PCT` | 0.02 | 半宽下限 |
| `PRESSURE_STAGE_HIGH_ATR` | 1.5 | 阶段新高窗口 |

### 9.7 突破与查询

**静态 locked**  
- **突破**：仅追加 `broken_idx`，不改 P/zone  
- **查询 @ t**：`confirm_idx ≤ t`；可选未突破、可选 `P > close[t]`

**动态 dynamic**  
- 查询 @ t：取当前 **持仓段**（`open_idx ≤ t ≤ close_idx`）中 **`bar_idx == t`** 的快照  
- 画线：连接持仓期内各日 `P_t` →「均线压力轨」

### 9.8 动态压力位（持仓期均线）

**用途**：**有持仓的每个交易日**，标注当日上方均线阻力（如 20 日线）；空仓不算。与结构 **locked** 压力位互补。

**计算（因果）**  

```
若 bar t 空仓 或 MA 预热不足 → 不产出
否则 P_t = MA_N(close; end=t)    # 只用 close[0..t]
zone_t = [P_t − band_t, P_t + band_t]
confirm_idx = t                  # 当日收盘后可落库
```

**生命周期**  

| 阶段 | 行为 |
|------|------|
| 空仓 | 无动态压力位 |
| `open_idx ≤ t ≤ close_idx` | 每个持仓交易日一条快照，**不改历史日** |
| 已平仓 | 该 `position_id` 段停止；下次开仓起新段 |

**默认参数（草案）**

| 参数 | 默认 | 含义 |
|------|------|------|
| `DYN_MA_PERIOD` | 20 | 如 20 日线 |
| `DYN_MA_MIN_BARS` | 20 | 满 N 根才输出 |
| `DYN_BAND_ATR` | 0.3 | 均线压力带半宽（ATR 倍数） |
| `DYN_ONLY_WHILE_HOLDING` | true | 仅持仓期间计算 |

**与静态压力位关系**  

- 同一图上：**locked** 画水平带（结构阻力）；**dynamic** 画 **持仓期内** 折线（均线轨）  
- 规则可同时引用：「接近 locked 压力位」与「收盘低于 dynamic MA20 压力」  
- 对外文案统一：「压力位」，实现用 `mode` 区分
