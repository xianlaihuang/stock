## 2. 共用框架：打分、成交与决策流水线

### 2.1 数据与预热

- 分析从 K 线索引 **`start_offset = 60`** 开始（前 60 根只用于指标预热）。
- 每条规则预计算为布尔序列：`buy_signals_pre[name][i]`、`sell_signals_pre[name][i]` 等。
- 买入前全局过滤（V2/V3/V4 均会调用，V4 对部分必买豁免）：
  - **至下一前高空间** ≥ 3%：`MIN_UPSIDE_TO_NEXT_PRIOR_HIGH_PCT = 0.03`
  - MACD/价升量增须配合形态或突破：`apply_macd_vp_requires_pattern_breakout_buy`
  - 刺透须共振：`apply_piercing_requires_confluence_buy`

### 2.2 加权得分（PRD 分组 cap）

对当日触发的规则名集合 \(T\)，按组求和后 cap，再按规则分摊：

```
对每个规则名 r：
  组 g = PRD_BUY_RULE_GROUP[r] 或 PRD_SELL_RULE_GROUP[r]
  组内原始和 S_g = Σ weight[r]（r 属于该组且 r ∈ T）
  有效组贡献 = min(S_g, V2_GROUP_SCORE_CAP)   # cap = 0.38
  buy_score / sell_score = 组贡献之和（每条规则按组内比例分摊）
```

### 2.3 成交与收益

| 参数 | 值 |
|------|-----|
| 执行模型 | `V2_EXECUTION_MODEL = 'next_open'`：信号日 `i` → 成交在 `open[i+1]`（末根用 `close[i]`） |
| 单边滑点 | `V2_SLIPPAGE_EACH_SIDE = 0.0005`（0.05%） |
| 往返佣金 | `V2_COMMISSION_ROUNDTRIP = 0.0008` |

**净收益率**（单笔 B→S）：

```
entry_in = entry_raw × (1 + 0.0005)
exit_out = exit_raw  × (1 - 0.0005)
net_ret  = (exit_out / entry_in) - 1 - 0.0008
```

### 2.4 每日决策优先级（持仓中 `last_signal == B`）

```
1. ATR 止损（low ≤ 入场价 - 2.5×ATR14）→ S
2. 时间止损（持仓 ≥ 55 个交易日）→ S
3. V4 专有：激进底破 V 底 / 连 2 日破 MA5 / 黄金坑 MA20 逻辑 → S
4. V4 专有：看跌形态（黄昏星/十字/射击之星）待破 MA5 → S
5. 必卖规则命中 → S（confidence=1）
6. 计数卖：触发卖出规则数 ≥ N 且 sell_score > buy_score → S
   - 默认 N=2；浮亏 ≤ -5.5% 或自高点回撤 ≤ -4.5% 时 N=1
7. 空仓时：必买 → B；或 buy_score > sell_score 且组合门控通过 → B（多走 pending）
```

**最小持有**：默认 `min_hold = 2` 根 K 线；高波动日（ATR/close > 历史中位数）为 `3`，买入置信门槛 `0.58`。

**非必买/非计数卖**：相邻同向信号间隔须 ≥ `min_hold`；必卖/止损等可打破间隔。

### 2.5 买入确认（Pending，V2/V3 为主）

信号日记录 `pending_buy`，确认窗口：

| 类型 | 确认条件 |
|------|----------|
| 默认 | T+1 或 T+2：`close > open` 且 `close > close[信号日]` |
| 双针探底（V4） | T+1：`close > open` 且 `close > MA5` |
| 看涨 K 线形态 | 仅 **下一交易日** 阳线（收盘>开盘） |
|  floor | 确认期间 low 不得跌破 `max(min(low[信号-5:信号+1]), close[信号] - 2.5×ATR)` |

**缩量确认**：确认日量 < 信号日量时，须 `close[确认] > close[信号]`。

## 3. V2 规则全集

> 实现文件：`backend/dynamic_weight_engine.py`（`@_register_buy` / `@_register_sell` / `@_register_mandatory_*`）

### 3.1 买入限制（触发则当日不可 B）

| 规则名 | 条件 |
|--------|------|
| 未突破5日线不买 | `close ≤ MA5` |
| 阴线不买 | `close ≤ open` |

### 3.2 必买（Mandatory Buy）

| 规则名 | 核心条件 |
|--------|----------|
| **W底突破** | 两低点间隔 8~80 根、价差 ≤5%；中间反弹 ≥3%；突破颈线且放量或收涨 |
| **头肩底突破** | 头肩底形态；收盘突破颈线（20 根内） |
| **V反底部** | 见 [§6.2 经典 V反底部（V2）](#62-经典-v反底部v2) |
| **旗形突破** | 旗杆涨 ≥5%；旗形整理；突破旗杆高点；量 > 1.3×5 日均量 |

### 3.3 加权买入

| 规则名 | 核心条件 |
|--------|----------|
| MA金叉 | MA5 上穿 MA20 |
| MACD金叉 / 底背离 | 金叉或价低量高背离；**须**同日有形态/突破/必买 |
| RSI超卖上穿 | RSI 上穿 30 |
| KDJ金叉 | K 上穿 D 且 K<20 |
| 看涨K线形态 | TA-Lib：看涨吞没、晨星、锤头、刺透、晨十字、倒锤头（先命中先返回） |
| 价升量增 / 放量突破高点 / 地量地价 | 量价条件；MACD/价升量增需形态共振 |
| 趋势线支撑 | 30 日线性回归斜率>0，收盘贴近趋势线 |
| 双针探底 | 见 [§6.4 双针探底](#64-双针探底) |
| 横盘整理向上突破 | 20 日箱体振幅/中价 ≤7%；突破上沿；阳线；量 ≥1.2×5 日均量 |
| 黄金坑 | 10 日振幅 ≥10%；价在 MA5 上、MA20 下；与 MA20 间距 ≥3 元或 ≥3%；坑口后放量阳线 |
| N字形突破 | 须在 W/V 右侧背景下；见 [§6.5 N 字形](#65-n-字形) |

**加权买入组合门控** `buy_weighted_combo_gate_ok`：

- 单独允许：`看涨K线形态`、`横盘整理向上突破`、`黄金坑`、`双针探底`、或 **唯一** `N字形突破`
- 否则：同日 ≥2 条加权买入规则

### 3.4 必卖（Mandatory Sell）

| 规则名 | 核心条件 |
|--------|----------|
| M顶跌破 | 双顶 ±2%；跌破顶部 98% |
| V反顶部 | 见 [§6.3 V 反顶部](#63-v-反顶部) |
| 旗形跌破 | 旗杆跌 ≥5%；跌破旗杆低点；放量 |
| 均线趋势打破必卖 | 沿 MA5/10/20 主升后有效跌破该均线（支撑比例 ≥70%） |
| 高位放量阴线看跌 | 进入历史压力带 + 当日阴线且量 > 1.5×前一日 |
| 一字涨停次日放量阴 | 前一日一字涨停 + 次日放量阴线 |

### 3.5 加权卖出（V2 计数卖用，以下 3 条在 V2 被剔除）

| 规则名 | V2 是否计数 | 核心条件 |
|--------|-------------|----------|
| MA死叉 / MACD死叉 / 顶背离 / RSI / KDJ | 是 | 标准技术指标 |
| 看跌K线形态 | 是 | 看跌吞没、黄昏星、上吊线、乌云盖顶、黄昏十字、射击之星；**黄昏星+前阳+缩量** 可剔除 |
| 价跌量增 / 放量跌破低点 / 天量天价 | 是 | 量价 |
| 趋势线阻力 / 头肩顶跌破 | 是 | 形态 |
| 均线趋势打破 | 是 | 加权版（非必卖阈值） |
| 沿均线主升破位 | 是 | 18 日内判定主导均线(5/10/20)，首次跌破 |
| **横盘整理向下突破** | **否（V3+）** | 20 日窄幅箱体向下突破 + 阴线 + 放量 |
| **历史缺口阻力** | **否（V3+）** | 见 [§6.6 上方缺口](#66-上方缺口) |
| **接近前高巨量阴增** | **否（V3+）** | 压力带 + 近 5 日 ≥3 阴 + 放量阴 |

### 3.6 V2 全局买入否决

| 过滤器 | 公式/条件 |
|--------|-----------|
| 上影线否决 | 上影 > 实体×0.75，或（振幅足够时）上影 ≥ 振幅×40% |
| 非下跌趋势 | `MA20↓` 且 `close < MA20` 时禁 B（**例外**：黄金坑、双针探底） |
| 突破前高否决 | 至下一前高空间 < 3% |
| idx < 20 或 MA20 无效 | 不出信号 |

## 6. V 形 / W 形结构计算（重点）

### 6.1 W 底右侧

**代码**：`strategy_vw.detect_w_right_bottom_events`（V3/V4 相同逻辑；V4 用 high 作瓶口）

**步骤**：

1. **筛选摆动低点**（基于 **close**）  
   - 在窗口 `[i-8, i+4]` 内 `close[i]` 为最小  
   - 相邻低点至少间隔 **5** 根  

2. **双底配对** `idx1 < idx2`  
   - 间隔：`8 ≤ idx2 - idx1 ≤ 80`  
   - 两底价差：`|close[idx1]-close[idx2]| / max(p1,p2) ≤ 5%`  

3. **瓶口（颈线）**  
   - `neck = max(high[idx1 : idx2+1])`（V3/V4；V2 必买 W底突破用 **close** 区间 max）  
   - 反弹幅度：`(neck - max(p1,p2)) / max(p1,p2) ≥ 3%`（`W_NECK_MIN_RISE = 0.03`）  

4. **右侧确认日 `entry`**  
   - 自 `idx2+1` 起，首根满足 `close[r] > close[idx2]`，且 `r - idx2 ≤ 10`  

5. **输出**  
   - `entry`：右侧确认 K 线索引（**即 V3/V4「W底右侧」必买日**）  
   - `neck`：瓶口价（high 最大）  
   - `stop_ref`：`min(low[idx1], low[idx2])`  

**去重**：相邻 entry 间隔 < 5 根则丢弃后者。

---

### 6.2 V 反右侧（V3）

**代码**：`strategy_vw_bottle_backtest.detect_v_right_bottom_events`

1. 在 `[idx-15, idx]` 内找 **low** 最低点 `low_idx`（须 `low_idx < idx` 且不太远）  
2. **左侧参考高**：`left_ref = max(high[low_idx-5 : low_idx])`  
3. **跌深**：`(left_ref - low[low_idx]) / left_ref ≥ 5%`（默认 `drop_min=0.05`）  
4. **右侧**：`low_idx+1` 起 10 根内，首根  
   - `(close[r]-low[low_idx])/low[low_idx] ≥ 2%`  
   - `close[r] > close[r-1]`  
5. 该根 `r` 为 **V反右侧** 必买日  

---

### 6.3 V 反右侧（V4 — 曲线化）

**代码**：`v4_structure_curves.V4StructureRegistry` → `strategy_vw.detect_v_right_bottom_events`

#### 6.3.1 四条价格曲线

```
avg[i] = (open[i] + high[i] + low[i] + close[i]) / 4
```

瓶口、前高用 **high**；V 左侧跌深用 **avg**（**不用** high−low）。

#### 6.3.2 ATR 摆动点

- `ATR(14)` 全序列  
- 在 **high** 上找摆动高点，在 **low** 上找摆动低点（`scipy.signal.find_peaks`）  
- 显著性：`prominence = max(median(ATR)×0.55, close×0.008)`  
- 最小间隔：`distance = 3` 根  

#### 6.3.3 识别 V 形

对每个摆动低点 `bi`（V 底候选）：

1. **左肩起点** `left_start`  
   - 取 `bi` 之前最近一个摆动 **高点** 的索引；若无，则 `bi - 60`  

2. **瓶口（high 曲线）**  
   ```
   neck = max(high[left_start : bi])    # 含 bi 当日
   ```

3. **V 底价格**（止损参考）  
   ```
   bottom_price = low[bi]
   ```

4. **左侧跌深（均价曲线，门槛 4%）**  
   ```
   peak_avg = max(avg[left_start : bi+1])
   drop_pct = (peak_avg - avg[bi]) / peak_avg
   要求：drop_pct >= V_DROP_MIN (= 0.04)
   ```

5. **右侧确认日 `right_entry`**  
   - 扫描 `r ∈ (bi+1 .. min(bi+60, bi+120))`  
   - 首根满足：  
     ```
     (close[r] - bottom_price) / bottom_price >= V_BOUNCE_MIN (= 0.02)
     close[r] > close[r-1]
     ```

6. **嵌套 V**  
   - 小 V 落在大 V 的 `[left_peak, bottom]` 区间内 → 标记 `parent`  
   - 发信号时优先 **最外层** V（`parent is None`）  

7. **输出事件**（`to_v_right_events`）  
   - `entry` = `right_entry` → **V反右侧必买日**  
   - `neck`, `stop_ref` = `bottom_price`  

**与 V3 的核心差别**：

| 项目 | V3 | V4 |
|------|----|----|
| 瓶口 | 固定 `low_idx` 前 5 根 high max | 上一摆动高点 → V 底 的 high max |
| 跌深 | high 低点，约 5% | **avg** 曲线，**4%** |
| 右侧窗口 | 10 根 | 最多 60 根（扫描上限 120） |

---

### 6.4 双针探底

**前提**：信号日落在某次 V/W **左侧底** 附近（`left_idx` 前 1 根 ~ 后 4 根），且左侧跌深 ≥ **4%**。

**形态**：

- **第一针**：信号日之前某根下影线针（下影/实体 ≥ 0.8）  
- **第二针**：信号日阳线，下影/实体 ≥ 0.8  
- 两针低点贴近 V/W 底且相差 < **0.5%**  

**买入确认**：

- V2/V3：通用 pending 阳线 + 高于信号日收盘  
- V4：T+1 `close > open` 且 `close > MA5`  

---

### 6.5 N 字形

**前提**：`find_vw_right_context_before` — 信号日前 **60** 根内存在 W/V **右侧** 事件 `entry`。

```
prior_high = max(high[entry : signal_idx])           # 右侧至信号日 high
prior_before = max(high[entry : signal_idx-1])      # 突破参照
```

条件概要：

1. 曾触及瓶口：`max(high[entry:idx]) >= neck × 0.998`  
2. 回踩腿低点在 prior_high 的 **88%~98%**，且靠近 neck  
3. 信号日突破：`high[idx] > prior_before × 1.001`（W 略严）  
4. 成交量 ≥ **1.12×** 5 日均量  

---

### 6.6 上方缺口

**组成**：由 **n 日** 与 **n+1 日** 两根 K 线形成（实现里右棒索引为 `n`）：

```
缺口成立：low[n-1] > high[n]       # 左棒最低 > 右棒最高；上沿 = low[n-1]，下沿 = high[n]
未回补：自 n 至当前（含），max(high[n:idx+1]) < low[n-1]
```

**历史缺口阻力**（加权卖，V3/V4）：

- 最近一处未回补缺口（回看 200 根，缺口后至少 5 根）  
- `high[idx] ≥ gap_top × 0.985`，当日阴线，且长上影 ≥35% 或 `close < gap_top × 0.995`  

---

### 6.7 经典 V反底部（V2）

**代码**：`buy_v_reversal_bottom`（仅 V2 必买标签 **V反底部**）

与 V3「V反右侧」不同，强调：

- 近 20 日内 **low** 最低点，且低点在 10 根以内  
- 跌深 5%、反弹 2%、收复左侧参考 ×98.5%  
- MA20 回踩确认、缩量过滤等（`V_REV_BOTTOM_*` 常量）  

---

### 6.8 V 反顶部（必卖）

**代码**：`sell_v_reversal_top`

- 过去 20 根（**不含当日**）的 high 最大值为峰  
- 左侧涨 ≥7%，距峰回落 ≥3%  
- 当日阴线，收盘 < 峰×98%，量能不过度萎缩  

---

### 6.9 V4 激进底部三条（摘要）

均在 V 左区 `low_idx ± (15, 8)` 扫描，共用 **V 左上下文**（`iter_v_left_bottoms`，跌深 ≥4% avg）。

| 规则 | 关键公式 |
|------|----------|
| 金针 | 下影/振幅 ≥1/3；下影 ≥0.75×实体；上影 ≤45% 振幅；`(MA20-high_entry)/high_entry ≥ 3%` |
| 大低开阳 | `open ≤ prev_close×(1-1%)`；阳线实体/振幅 ≥45%；MA20 空间 ≥2% |
| 止跌 | 下跌动能后半段均值 < 前半×0.88；均价止跌；`close>MA5`；MA20 空间 ≥3% |
| V4V反吞没 | 吞没阳线；实体/振幅 ≥52%；实体 ≥1.05×前日实体；次日阳线买入 |

**止损价**：V 底区间 `min(low)` → 收盘跌破则 **V反激进底破V底止损**。

### 6.10 瓶口半仓回测（非 B/S 标）

`strategy_vw.simulate_trade` / `strategy_vw_bottle_backtest` 中：

- 入场：entry 日收盘 **全仓** 买入  
- 触及 `neck×0.998`：**卖出 50%**  
- 突破 `neck×1.008` 后延长持有；否则跌破 `neck×0.992` 或超时清仓  

此逻辑用于 **V/W 轨道回测 API**，**不**改变 `analyze_signals_*` 的 B/S 配对（仍为全仓一次 B、一次 S）。

## 7. 权重优化与组合模拟

### 7.1 权重迭代（`calculate_weights_v2`）

1. **Layer1**：在高低点锚点 ±90 日窗口统计规则命中奖励/惩罚  
2. **迭代**（最多 100 轮，连续 3 次收益变化 <0.1% 收敛）：  
   - `_mark_bs_points` 生成信号 → 验证买卖规则有效性 → 更新权重  
3. **择优**：  
   - 默认：`U = Σr − 0.55×MDD − 0.12×下行波动 − 0.00025×笔数 − 0.02×L1(w)`  
   - 可关闭效用，退回 **总收益率最大**

### 7.2 组合模拟

- V2/V3：`engine_v2._portfolio_sim_from_paired` — 每笔 S 的 `return_pct` 全仓复利

初始资金展示默认 **100 万**（前端 `portfolio_sim`）。
