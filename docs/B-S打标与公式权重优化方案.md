# 公式权重与 B/S 打标逻辑优化方案

**文档性质**：策略与数学优化建议（非投资建议）  
**编制视角**：资深操盘手实务 + 量化/风险管理数学框架  
**对齐代码基线**：`engine_v2.py`（`calculate_weights_v2`、`_mark_bs_points`、`analyze_signals_v2`）、`dynamic_weight_engine`（统一动态权重路径）  
**核心目标**：在可控风险下提升风险调整后收益，并降低过拟合与执行落差  

---

## 1. 执行摘要

当前系统以「多规则加权打分 + 必买/必卖/计数卖 + 阳线确认」完成 B/S 打标，权重层以「锚点窗口计数 + 迭代区间内奖惩 + 总收益率择优」为主。该设计在**可解释性**与**工程可实现性**上较好，但在**风险维度显式化**、**目标函数与交易现实的一致性**、**样本外稳健性**三方面仍有显著提升空间。

本方案建议将优化目标从单一的「累计收益率最大化」升级为**多目标效用函数**（收益 − 风险惩罚 − 换手惩罚 − 复杂度惩罚），并在打标层引入**波动率状态、ATR 止损语义、流动性/极端日过滤**与**更严谨的确认价**。实施上可按 P0/P1/P2 分期落地，避免一次性大改导致回归不可控。

## 2. 现状简评（操盘手 + 数学双视角）

### 2.1 权重与迭代（`calculate_weights_v2`）

| 维度 | 现状要点 | 风险与偏差 |
|------|----------|------------|
| 优化目标 | 以配对 B→S 的收盘价收益之和最大为「最佳」 | 单笔大盈可掩盖高波动与深回撤；与「稳健复利」目标不一致 |
| 奖惩刻度 | 收益按 10% 档位离散加 reward | 粗粒度导致梯度信息丢失；对极端行情敏感 |
| 第一层锚点 | 高低点 + 抛物线过滤 + ±90 日窗口命中 | 锚点选择依赖历史极值结构，在震荡市可能过密或过稀 |
| 收敛 | 连续 3 次总收益变化 < 0.1% | 可能停在平台区而非全局更优；且未校验样本外 |

### 2.2 B/S 打标（`_mark_bs_points`）

| 维度 | 现状要点 | 风险与执行落差 |
|------|----------|----------------|
| 买入确认 | 信号日次日阳线且收盘高于信号日收盘 | 未规定成交价：实盘常为信号次日开盘或突破价，存在**跳空与滑点** |
| 卖出 | 必卖/计数卖优先 | 计数卖依赖「条数≥2」，在噪声市可能滞后；缺少**硬止损**语义 |
| 趋势过滤 | MA20 下行且价在 MA20 下则禁买 | 单一均线状态对趋势/震荡切换描述不足 |
| 最小持有 | `min_hold=3`（高波动日 `V2_HIGH_VOL_MIN_HOLD=4`） | 抑制震荡段过密 B/S；成本与冲击更贴近实盘 |
| 置信度 | `buy_score/(buy_score+sell_score)` 阈值 0.55 | 在规则数多时，分母结构未反映**规则相关性**（多重共线） |

### 2.3 与「最大收益 + 风险控制」的差距（一句话）

当前逻辑更接近「在历史标签上拟合一套可解释的加权投票」，尚缺「**把回撤、波动、换手与尾部风险写进目标函数与打标约束**」这一环。

## 3. 优化总原则

1. **目标可微或可分**：便于从「拍阈值」过渡到「网格/贝叶斯/约束优化」。  
2. **训练与验证分离**：时间序列上必须 walk-forward 或滚动验证，避免同一窗口既选权重又评权重。  
3. **风险优先序**：先限制最坏情况（尾部、最大回撤、连续亏损），再抬收益。  
4. **执行一致性**：打标价、确认价、止损价与回测价三者规则统一，否则优化的是「幻影 alpha」。  

---

## 4. 数学层：目标函数与权重优化

### 4.1 建议的效用函数（标量化多目标）

设按序排列的配对交易收益为 \(r_1,\ldots,r_T\)（已含成本 \(c\) 的净收益），权益曲线为 \(W_t=\prod_{i\le t}(1+r_i)\)。

定义以下可计算量：

- **年化或样本内总收益**：\(G = \sum r_i\) 或几何均值。  
- **下行波动**（Sortino 分母）：\(\sigma_d = \sqrt{\frac{1}{T}\sum \min(0, r_i-\tau)^2}\)，\(\tau\) 可为 0 或无风险利率。  
- **最大回撤**：\(MDD = \max_s \frac{\max_{t\le s}W_t - W_s}{\max_{t\le s}W_t}\)。  
- **换手惩罚**：以信号变更次数或有效持仓天数倒数度量。  

**建议效用（示例）**：

\[
U = G - \lambda_1 \cdot MDD - \lambda_2 \cdot \sigma_d - \lambda_3 \cdot N_{\text{turnover}} - \lambda_4 \cdot \|w\|_1
\]

其中 \(w\) 为买入或卖出规则权重向量；\(\lambda_1\gg\lambda_2\) 体现「亏得少比赚得略多更重要」的操盘偏好。

**实现提示**：在现有「总收益率择优」分支上，将 `if total_return > best_return` 改为 `if U > best_U`，并同步记录 `best_mdd`、`best_sortino` 便于报表。

### 4.2 权重正则化与稀疏化

- **熵正则**：鼓励权重分布不过度集中，降低单规则黑天鹅依赖：在 \(U\) 中减 \(\lambda_H \sum w\log(nw)\)。  
- **L1 惩罚**：推动弱规则权重归零，提升可解释性与稳定性。  
- **最小样本约束**：对触发次数低于 \(N_{\min}\) 的规则，在优化中直接屏蔽或施以强惩罚（与动态引擎中 `_graduated_penalty` 思想一致，可统一到 V2）。

### 4.3 时间截面验证（必做）

- **滚动训练窗**：例如用过去 \(L\) 根 K 线估权重，在未来 \(H\) 根上只评估 \(U\)，向前滚动。  
- **选参准则**：以验证集 \(U\) 均值或中位数选权重，而非全样本单次最优。  

此举对「最大收益」在数学上看似次优，在实务上常**提升样本外收益与降低回撤**。

### 4.4 第一层锚点权重的稳健化（可选）

对 S/B 锚点附近的规则命中计数，可改为：

- **核权重**：以距离锚点的天数为核 \(K(d)\) 加权命中，减少窗口边界跳变。  
- **Bootstrap**：对锚点集重抽样估计规则重要度方差，剔除高方差弱规则。

## 5. 打标层：B/S 逻辑优化（贴近实盘与风控）

### 5.1 成交价模型（消除执行幻觉）

| 信号类型 | 建议回测默认价 | 说明 |
|----------|----------------|------|
| 确认买入 B | 确认日 **开盘价** 或 `min(开盘, 信号日高点+ε)` | 与「突破确认」语义一致 |
| 卖出 S | 信号日 **收盘价** 或次日开盘（可配置） | 必卖类可偏保守用更差一侧 |

在 `_verify_buy_effectiveness` / `_calculate_total_return` 中与打标使用同一套价格规则，否则优化目标与实盘脱节。

### 5.2 硬止损与「时间止损」（风控核心）

在持仓状态下增加**独立于打分**的平仓条件（满足任一即 S 或强制减仓标记）：

- **ATR 止损**：买入价下方 \(k \cdot ATR(n)\) 破位（常用 \(n=14, k=2\sim3\)）。  
- **最大持有天数**：防止趋势失效后长期套牢。  
- **结构化止损**：前低/颈线跌破（可与现有形态规则合并优先级）。

数学上这是在策略路径上增加**吸收态边界**，可显著降低 \(MDD\) 的尾部贡献。

### 5.3 波动率状态门控（Regime）

将市场分为低/高波动（例如用 \(ATR/close\) 或 realized vol 分位数）：

- **高波动**：提高 `confidence_medium`、提高 `min_hold`、或要求更强必买才开仓。  
- **低波动**：允许略放宽，但防范假突破（结合成交量确认）。

避免全周期同一阈值导致的「震荡市过度交易 / 趋势市信号不足」。

### 5.4 分数合成：从线性加到「相关结构感知」

当前 `buy_score = \sum w_i I_i` 假设规则独立。建议渐进式改进：

1. **分组 caps**：同类指标（如 MACD 与均线）共享上限，避免同源重复计数。  
2. **二阶项或逻辑回归**：用少量参数学习规则交互，仍以稀疏约束控制过拟合。  

### 5.5 卖出侧：不对称优化

实盘常见「卖飞」与「套牢」不对称心理，数学上可用：

- **更快响应亏损**：当浮动盈亏 < −\(x\%\) 时，降低计数卖阈值或提升 sell_score 权重。  
- **盈利保护**：达到 \(y\%\) 后启用移动止损（吊灯止损等）。

## 6. 与统一动态权重路径的协同（若双轨并存）

- **目标对齐**：`optimize_weights_v2` 的深度扫描 + 质量分与 V2 迭代可共用**同一套回测价与成本模型**，避免两套逻辑产出不可比信号。  
- **缓存与持久化**：进程内 `_weight_cache` 不利于多实例与重启恢复，建议统一到 Mongo 并带**版本号与训练截止日**，便于审计。

## 7. 实施路线图（建议）

| 优先级 | 内容 | 预期收益 | 复杂度 |
|--------|------|----------|--------|
| P0 | 回测统一成交价 + 固定双边成本/滑点 + 效用 \(U\) 替代单一总收益 | 显著降低执行幻觉与虚高收益 | 中 |
| P0 | Walk-forward 或滚动验证选权重 | 样本外稳健性 | 中高 |
| P1 | ATR/时间止损接入打标与奖惩 | 回撤与尾部风险 | 中 |
| P1 | 波动率门控阈值 | 减少震荡市无效换手 | 中 |
| P2 | 规则分组 cap / 权重正则 | 降低共线与过拟合 | 中高 |
| P2 | 多周期确认（仅日线触发 + 周线趋势过滤） | 趋势段质量 | 高 |

## 8. 验收指标（上线前后对比）

建议在回测报告中固定输出：

1. **净收益**（扣成本）、**年化收益**、**最大回撤**、**Calmar**、**Sortino**。  
2. **胜率**、**盈亏比**、**平均持仓天数**、**年化换手**。  
3. **最大连续亏损笔数**、**最差单月收益**。  
4. **参数敏感性曲线**（阈值 ±20% 时 \(U\) 变化）。

验收标准示例：在相同数据段上，P0 完成后 \(MDD\) 下降或 Sortino 提升，且 walk-forward 验证集 \(U\) 不劣于基线。

## 11. 产品反馈落地（形态与前端）

### 11.1 必卖「V反顶部」（`dynamic_weight_engine.sell_v_reversal_top`）

针对个案复盘（如未形成清晰 V 顶仍触发必卖）收紧形态语义：

- 窗口内 **20 日高点不含当日**，避免当日上影与「见顶回落」混淆。  
- 高点须为 **±2 根 K 内的局部最高价**，过滤震荡中的伪峰。  
- 左侧低点至峰值的涨幅阈值 **≥7%**；自峰值至当日的 **最低价比峰值回落 ≥3%**，且 **收盘低于峰值约 2% 以上**。  
- 仍要求阴线（`close[idx] < close[idx-1]`）、放量不低于近 5 日均量 0.8 倍；峰值距信号日 **至少 3 根、至多落在窗口前半**（与 `recent_high_idx < idx - 10` 一致）。

### 11.2 智能分析 B/S K 线图交互（`frontend/index.html`）

- **缩放**：仅通过 **↑ / ↓** 键调整 `dataZoom` 范围（与主 K 线图键盘逻辑一致）；关闭底部 **slider** 与画布 **拖拽平移**，避免鼠标缩放/拖动与「仅键盘缩放」需求冲突。  
- **Tab 切换**：任意 Tab 切换时若存在信号图实例，将 `dataZoom` **重置为默认**（最近约 50 根可见窗口），避免跨页签保留畸变视窗。
